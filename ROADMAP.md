# Roadmap

이 문서는 구현 순서를 고정하기 위한 로드맵이다. 목표는 모든 것을 한 번에 만들지 않고, 문서와 실행 가능한 기능을 함께 전진시키는 것이다.

## 설계 원칙

- 먼저 parser가 안정적으로 동작해야 한다.
- 그다음 interpreter로 핵심 예제를 실행한다.
- 자연스러운 문법 확장은 v0.1 실행 기반이 생긴 뒤 진행한다.

## v0.1

목표: 문서 기준으로 작동하는 최소 실행기를 만든다.

포함:

- lexer
- parser
- AST
- 바인딩과 재대입
- 출력
- 조건문
- 반복문
- 함수 정의와 호출
- 목록과 레코드
- `의` 속성 접근
- 한국어 오류 메시지 초안
- 문법 회귀 테스트

완료 조건:

- `EXAMPLES.md`의 핵심 예제 대부분이 실행된다.
- parser snapshot test가 최소 30개 있다.
- 런타임 오류 메시지 예제가 문서와 대체로 일치한다.

## v0.2

목표: 문장형 경험을 더 자연스럽게 확장한다.

후보 기능:

- 비교 표현의 한국어형 확장
- `목록의 길이` 같은 내장 속성 lowering
- `각` 기반 반복문
- 간단한 타입 주석
- REPL
- 포매터 초안

## v0.3

목표: 도구성과 성능을 강화한다.

후보 기능:

- 정적 검사 계층
- 바이트코드 또는 LLVM/Cranelift 백엔드
- LSP
- 패키지/모듈 구조 초안
- 표준 라이브러리 확장

## 장기 연구 과제

- 조사 생략 허용 범위 연구
- 자연어형 수식 파싱
- 한국어 친화 진단 자동 수정
- 교육용 모드와 일반 모드 분리
- 음성 입력/낭독 친화 문법

## 권장 진행 순서

1. 문서 확정
2. lexer/parser
3. parser 테스트 고정
4. interpreter
5. 진단 품질 개선
6. 문법 확장
