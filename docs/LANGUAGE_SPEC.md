# Language Specification

이 문서는 지음(ziium)의 v0.1 표면 문법을 정의한다. v0.1은 규범적 초안이며, 구현과 테스트는 이 문서를 기준으로 작성한다.

## 1. 범위

v0.1은 아래를 다룬다.

- 어휘 규칙
- 들여쓰기 블록
- 바인딩과 재대입
- 출력
- 조건문과 반복문
- 함수 정의와 호출
- 기본 표현식
- 속성 접근
- 런타임 값 종류

다음 항목은 v0.1 범위 밖이다.

- 모듈 시스템
- 사용자 정의 타입 선언
- 패턴 매칭
- 제네릭
- 매크로
- 암시적 주어 복원

## 2. 규범 용어

이 문서의 "해야 한다"는 필수 요구사항이다. "권장한다"는 구현 권고사항이다.

## 3. 핵심 원칙

### 3.1 문장 종결

한 줄은 기본적으로 하나의 문장을 이룬다. 문장은 줄바꿈으로 종료된다. 단, 괄호 `()`, 대괄호 `[]`, 중괄호 `{}` 안에서는 줄바꿈이 문장을 종료하지 않는다.

완료된 문장 뒤에는 선택적으로 마침표 `.`를 붙일 수 있다. 마침표는 문장 종결 보조 기호이며, `이다`, `바꾼다`, `출력한다`, `돌려준다`, 독립 호출 문장 뒤에서만 허용한다.

블록을 여는 줄에는 마침표를 붙이지 않는다. 즉 `...이면`, `...인 동안`, `... 함수는 ... 받아`, `아니면` 뒤에는 `.`를 허용하지 않는다.

### 3.2 들여쓰기 기반 블록

블록은 들여쓰기로 구분한다. v0.1에서 탭 문자는 허용하지 않는다. 스페이스 2칸 또는 4칸을 권장하지만, 구현은 "같은 블록에서 일관된 들여쓰기 깊이"만 만족하면 된다.

### 3.3 엄격한 문법

v0.1은 자연어 추측을 하지 않는다.

- 조사가 필요한 문장에서는 조사를 생략할 수 없다.
- 조건식은 명시적으로 불리언 값을 만들어야 한다.
- 이름 해석은 렉시컬 스코프를 따른다.

## 4. 어휘 규칙

### 4.1 문자 인코딩

소스 파일은 UTF-8이어야 한다.

### 4.2 공백

- 스페이스와 줄바꿈은 공백으로 취급한다.
- 탭은 금지한다.
- 문자열 리터럴 내부 공백은 보존한다.

### 4.3 주석

`#`부터 줄 끝까지는 주석이다.

예:

```txt
나이는 20이다  # 현재 사용자 나이
```

### 4.4 식별자

식별자는 아래 규칙을 따른다.

- 첫 글자는 한글 음절, 영문자, `_` 중 하나여야 한다.
- 이후 글자는 한글 음절, 영문자, 숫자, `_`를 포함할 수 있다.
- 예약어와 완전히 같은 식별자는 사용할 수 없다.

예:

- `나이`
- `학생_수`
- `score`
- `_임시`

### 4.5 예약어

v0.1 예약어는 다음과 같다.

- `함수`
- `이다`
- `이면`
- `아니면`
- `인`
- `동안`
- `받아`
- `받지`
- `않아`
- `출력한다`
- `돌려준다`
- `바꾼다`
- `참`
- `거짓`
- `없음`
- `그리고`
- `또는`
- `아니다`

### 4.6 조사 토큰

v0.1은 아래 조사를 의미 있는 문법 요소로 처리한다.

- 주제: `은`, `는`
- 주어: `이`, `가`
- 목적어: `을`, `를`
- 소유/접근: `의`
- 방향: `로`, `으로`

구현은 `사람의` 같은 표면 형태를 `사람` + `의`로 분리할 수 있어야 한다.

### 4.7 리터럴

지원 리터럴:

- 정수: `0`, `42`, `-7`
- 실수: `3.14`, `-0.5`
- 문자열: `"안녕"`
- 불리언: `참`, `거짓`
- 없음 값: `없음`
- 목록: `[1, 2, 3]`
- 레코드: `{ 이름: "철수", 나이: 20 }`

문자열은 v0.1에서 큰따옴표만 허용한다.

## 5. 값 모델

v0.1 런타임 값은 아래 종류를 가진다.

- 정수
- 실수
- 문자열
- 불리언
- 없음
- 목록
- 레코드
- 함수

조건식은 오직 불리언만 허용한다. Python 식의 truthiness는 지원하지 않는다.

## 6. 블록

블록은 아래 문장 뒤에 온다.

- `...이면`
- `아니면`
- `...인 동안`
- 함수 정의 헤더

예:

```txt
나이 > 19이면
  "성인"을 출력한다
아니면
  "미성년자"를 출력한다
```

## 7. 바인딩

### 7.1 문법

기본 바인딩 형식:

```txt
<이름>은 <표현식>이다
<이름>는 <표현식>이다
<이름>은 <표현식>이다.
<이름>는 <표현식>이다.
```

### 7.2 의미

- 현재 스코프에 새로운 이름을 만든다.
- 같은 스코프에 같은 이름이 이미 있으면 오류다.
- 우변 표현식은 먼저 평가된다.

예:

```txt
이름은 "철수"이다.
나이는 20이다.
목록은 [1, 2, 3]이다.
```

## 8. 재대입

### 8.1 문법

```txt
<이름>을 <표현식>로 바꾼다
<이름>를 <표현식>로 바꾼다
<이름>을 <표현식>으로 바꾼다
<이름>를 <표현식>으로 바꾼다
<이름>을 <표현식>로 바꾼다.
<이름>를 <표현식>로 바꾼다.
<이름>을 <표현식>으로 바꾼다.
<이름>를 <표현식>으로 바꾼다.
```

### 8.2 의미

- 이미 존재하는 바인딩을 갱신한다.
- 이름이 없으면 오류다.

예:

```txt
개수는 0이다
개수를 개수 + 1로 바꾼다
```

## 9. 출력

### 9.1 문법

```txt
<표현식>을 출력한다
<표현식>를 출력한다
<표현식>을 출력한다.
<표현식>를 출력한다.
```

### 9.2 의미

- 표현식을 평가한 뒤 표준 출력으로 보낸다.
- 출력 형식은 인터프리터/REPL 구현이 정한다.

## 10. 반환

### 10.1 문법

```txt
<표현식>을 돌려준다
<표현식>를 돌려준다
<표현식>을 돌려준다.
<표현식>를 돌려준다.
```

### 10.2 의미

- 함수 본문 안에서만 허용한다.
- 현재 함수 실행을 종료하고 값을 반환한다.

## 11. 조건문

### 11.1 문법

```txt
<표현식>이면
  <블록>

<표현식>이면
  <블록>
아니면
  <블록>
```

### 11.2 의미

- 조건 표현식은 불리언이어야 한다.
- `아니면` 블록은 선택 사항이다.

## 12. 반복문

### 12.1 문법

```txt
<표현식>인 동안
  <블록>
```

### 12.2 의미

- 조건은 반복마다 다시 평가된다.
- 조건 표현식은 불리언이어야 한다.

예:

```txt
인덱스는 0이다
인덱스 < 길이(목록)인 동안
  목록[인덱스]을 출력한다
  인덱스를 인덱스 + 1로 바꾼다
```

## 13. 함수 정의

### 13.1 매개변수가 있는 함수

문법:

```txt
<이름> 함수는 <매개변수 목록>을 받아
  <블록>
```

매개변수 목록은 쉼표로 구분한다.

예:

```txt
더하기 함수는 왼쪽, 오른쪽을 받아
  왼쪽 + 오른쪽을 돌려준다
```

### 13.2 매개변수가 없는 함수

문법:

```txt
<이름> 함수는 아무것도 받지 않아
  <블록>
```

예:

```txt
시작 함수는 아무것도 받지 않아
  "시작"을 출력한다
```

### 13.3 의미

- 함수는 정의 시 현재 렉시컬 환경을 캡처할 수 있다.
- v0.1은 이름 있는 함수 정의만 필수 지원한다.
- 함수 정의 헤더에서 조사는 `함수는`으로 고정한다.

## 14. 표현식

### 14.1 기본 표현식

- 식별자
- 리터럴
- 괄호 표현식
- 함수 호출
- 인덱싱
- 속성 접근

### 14.2 함수 호출

문법:

```txt
<식별자>(<인수 목록>)
```

예:

```txt
합은 더하기(3, 5)이다
```

함수 호출은 문장으로 단독 사용해도 된다. 이 경우 반환값은 버린다.

예:

```txt
시작()
추가(숫자들, 3)
```

### 14.3 인덱싱

문법:

```txt
<표현식>[<표현식>]
```

### 14.4 속성 접근

문법:

```txt
<표현식>의 <식별자>
```

예:

```txt
사람의 이름
설정의 포트
숫자들의 길이
```

속성 접근은 좌결합이다.

```txt
사용자의 주소의 도시
```

위 표현은 `(사용자의 주소)의 도시`로 해석한다.

v0.1 구현은 아래 명사형 속성을 추가로 지원한다.

- 목록의 `길이`
- 문자열의 `길이`

즉 `목록의 길이`는 `길이(목록)`과 같은 값으로 평가된다.

### 14.5 연산자

v0.1은 아래 연산자를 지원한다.

- 산술: `+`, `-`, `*`, `/`, `%`
- 비교: `==`, `!=`, `<`, `<=`, `>`, `>=`
- 논리: `그리고`, `또는`
- 단항: `-`, `아니다`

구현은 추가로 아래 built-in 단어 연산을 지원한다.

- `더하기`
- `빼기`
- `곱하기`
- `나누기`

예:

```txt
합은 7 더하기 8이다.
차는 10 빼기 3이다.
```

이 단어 연산들은 각각 `+`, `-`, `*`, `/`와 같은 우선순위로 평가된다.

### 14.6 우선순위

높은 쪽에서 낮은 쪽 순서:

1. 후위 연산: 호출, 인덱싱, 속성 접근
2. 단항 연산: `-`, `아니다`
3. 곱셈 계열: `*`, `/`, `%`
4. 덧셈 계열: `+`, `-`
5. 비교 계열: `<`, `<=`, `>`, `>=`
6. 동등성: `==`, `!=`
7. 논리곱: `그리고`
8. 논리합: `또는`

## 15. 레코드

레코드 리터럴은 중괄호로 쓴다.

```txt
사람은 { 이름: "철수", 나이: 20 }이다
```

키는 식별자 또는 문자열일 수 있다. 식별자 키는 문자열 키로 정규화된다.

## 16. 목록

목록 리터럴은 대괄호로 쓴다.

```txt
숫자들은 [1, 2, 3]이다
```

인덱스는 0부터 시작한다.

구현은 추가로 아래 built-in 키워드 메시지를 지원한다.

```txt
<목록>에 <값> 추가
<목록>에 <값> 추가.
```

이 형태는 `추가(<목록>, <값>)`와 같은 효과를 가진다.

## 17. 스코프

- 파일 최상위는 전역 스코프다.
- 함수 본문은 새 스코프를 만든다.
- 조건문과 반복문 블록은 새 블록 스코프를 만들지 않는다. v0.1에서는 단순화를 위해 함수만 새 지역 스코프를 만든다.
- 조건문 뒤에서 이름을 사용하려면 모든 분기에서 그 이름이 바인딩되어 있어야 한다.
- 반복문 본문에서 처음 바인딩한 이름은 반복문 뒤에서 자동으로 사용 가능해지지 않는다.

## 18. 내장 함수

v0.1 필수 내장 함수:

- `길이(값)`
- `문자열로(값)`
- `정수로(값)`
- `실수로(값)`
- `추가(목록, 값)`

`추가`는 목록 끝에 값을 추가하고 `없음`을 돌려준다.

추가로 아래 명사형 속성을 지원한다.

- `목록의 길이`
- `문자열의 길이`

추가로 아래 built-in 키워드 메시지를 지원한다.

- `<목록>에 <값> 추가`

## 19. 금지되는 표현

아래는 v0.1에서 금지한다.

- 조사 생략 바인딩: `이름 "철수"이다`
- 암시적 재대입: `이름은 "영희"이다`를 재대입 의미로 쓰기
- 참/거짓이 아닌 조건식
- 탭이 섞인 들여쓰기
- `의` 연산자 뒤에 식별자가 아닌 임의 표현식 쓰기
- 같은 줄에 여러 문장을 세미콜론으로 연결하기

## 20. 진단 원칙

오류 메시지는 최소한 아래를 포함해야 한다.

- 어디서 실패했는지
- 기대한 문장 형태가 무엇인지
- 가능한 수정 방향

자세한 원칙은 `ERRORS.md`를 따른다.
