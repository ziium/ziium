# Tasks

이 문서는 바로 수행 가능한 작업 단위를 정리한다. 각 항목은 AI 에이전트나 사람이 독립적으로 진행할 수 있어야 한다.

## 작성 규칙

각 작업은 아래 요소를 포함한다.

- 배경
- 해야 할 일
- 완료 조건
- 제외 범위

## 작업: 토큰 규격 초안 작성

### 배경

`GRAMMAR.ebnf`는 토큰 이름을 정의하지만, lexer 세부 규칙은 별도 정리가 필요하다.

### 해야 할 일

- 식별자, 리터럴, 예약어, 조사 토큰 표를 만든다.
- 붙은 조사 분리 예시를 15개 이상 수집한다.
- 탭 금지와 들여쓰기 토큰화 규칙을 명시한다.
- 산출물을 `TOKENS.md`로 정리한다.

### 완료 조건

- lexer 구현자가 별도 추측 없이 토큰 분리를 시작할 수 있다.

### 제외 범위

- 실제 lexer 코드 작성

## 작업: `의` 접근식 parser 테스트 추가

### 배경

`의`는 이 언어의 정체성에 직접 연결되는 핵심 후위 연산자다.

### 해야 할 일

- `사람의 이름`
- `사용자의 주소의 도시`
- `목록[0]의 이름`

위 형태를 parser golden test로 추가한다.
- 산출물 초안은 `PARSER_TEST_CASES.md`에 정리한다.

### 완료 조건

- 좌결합 구조가 snapshot으로 고정된다.

### 제외 범위

- interpreter 구현

## 작업: 바인딩/재대입 구분 구현

### 배경

`...은 ...이다`와 `...를 ...로 바꾼다`는 내부적으로 전혀 다른 문장이다.

### 해야 할 일

- parser가 두 문장을 다른 AST 노드로 만든다.
- 같은 스코프 재선언 오류를 정의한다.
- 미정의 이름 재대입 오류를 정의한다.

### 완료 조건

- 선언과 갱신 관련 테스트가 통과한다.

### 제외 범위

- 타입 시스템

## 작업: 조건문과 `아니면` 블록 구현

### 배경

분기문은 최소 실행기의 핵심이다.

### 해야 할 일

- `...이면` 블록 파싱
- 선택적 `아니면` 블록 파싱
- 들여쓰기 오류 진단 추가

### 완료 조건

- 중첩 조건문 예제가 파싱되고 실행된다.

### 제외 범위

- `아니고` 같은 추가 연결문

## 작업: 반복문 실행기 구현

### 배경

목록 처리와 누적 계산을 위해 반복문이 필요하다.

### 해야 할 일

- `...인 동안` 파싱
- 반복 조건 평가
- 반복 중 상태 갱신

### 완료 조건

- 카운터 예제가 실행된다.

### 제외 범위

- 컬렉션 기반 `for-each`

## 작업: 함수 정의와 호출 구현

### 배경

언어가 스크립트 수준을 넘으려면 함수가 필요하다.

### 해야 할 일

- 매개변수 있는 함수 정의
- 매개변수 없는 함수 정의
- 함수 호출과 인수 개수 검사
- `돌려준다` 처리

### 완료 조건

- `더하기`, `시작` 예제가 실행된다.

### 제외 범위

- 익명 함수

## 작업: 한국어 오류 메시지 템플릿 작성

### 배경

언어 경험은 오류 메시지까지 포함한다.

### 해야 할 일

- 파서 오류 10종
- 이름 오류 5종
- 속성 접근 오류 5종

각 오류에 대해 문장형 메시지 초안을 작성한다.

### 완료 조건

- `ERRORS.md` 예시와 구현 메시지가 연결된다.

### 제외 범위

- 자동 수정 기능
