# Error Message Guidelines

이 문서는 오류 메시지의 문체와 형식을 정의한다. 이 언어에서는 오류 메시지도 한국어 문장형 경험의 일부다.

## 목표

- 무엇이 잘못되었는지 즉시 알 수 있게 한다.
- 사용자가 기대 문법을 추측하지 않아도 되게 한다.
- 가능한 경우 수정 방향을 함께 제시한다.

## 원칙

### 1. 문제를 먼저 말한다

나쁜 예:

```txt
ParseError at line 3
```

좋은 예:

```txt
3번째 줄에서 조건문이 끝나지 않았습니다.
```

### 2. 기대한 형태를 짧게 보여준다

예:

```txt
`이면` 뒤에는 들여쓴 블록이 와야 합니다.
예:
  나이 > 19이면
    "성인"을 출력한다
```

### 3. 한국어 문체를 유지한다

- 지나치게 기계적인 축약형을 피한다.
- 영어 원문을 그대로 노출하지 않는다.
- 내부 구현 용어는 필요할 때만 괄호로 보조한다.

### 4. span 정보를 포함한다

가능하면 아래 정보를 제공한다.

- 줄 번호
- 열 번호
- 문제 구간 표시

## 오류 분류

- 어휘 오류
- 구문 오류
- 이름 오류
- 호출 오류
- 속성 접근 오류
- 실행 오류

## 예시

### 잘못된 들여쓰기

입력:

```txt
참이면
 "a"를 출력한다
```

메시지 예시:

```txt
2번째 줄 들여쓰기가 올바르지 않습니다.
`이면` 뒤 블록은 같은 깊이로 들여써야 합니다.
```

### 미정의 이름

입력:

```txt
이름을 출력한다
```

메시지 예시:

```txt
`이름`은 아직 정의되지 않았습니다.
먼저 `이름은 ...이다`처럼 값을 바인딩해 주세요.
```

### 재대입 대상 없음

입력:

```txt
점수를 10으로 바꾼다
```

메시지 예시:

```txt
`점수`를 바꿀 수 없습니다. 이 이름이 현재 스코프에 없습니다.
먼저 `점수는 ...이다`로 정의해 주세요.
```

### 조건식이 불리언이 아님

입력:

```txt
개수이면
  "있다"를 출력한다
```

메시지 예시:

```txt
조건식은 `참` 또는 `거짓`으로 평가되어야 합니다.
`개수 > 0이면`처럼 비교식을 써 보세요.
```

### 없는 속성 접근

입력:

```txt
사람은 { 이름: "철수" }이다
사람의 나이를 출력한다
```

메시지 예시:

```txt
`사람`에는 `나이` 속성이 없습니다.
사용 가능한 속성: `이름`
```

## 금지할 메시지 스타일

- 내부 enum 이름을 그대로 보여주는 메시지
- 영어 스택 트레이스만 출력하는 메시지
- "invalid syntax"처럼 정보가 거의 없는 메시지
- 수정 예시 없이 끝나는 파서 오류
